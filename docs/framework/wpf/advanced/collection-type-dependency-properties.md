# <a name="collection-type-dependency-properties"></a><span data-ttu-id="fb60f-102">Propriétés de dépendance de type collection</span><span class="sxs-lookup"><span data-stu-id="fb60f-102">Collection-Type Dependency Properties</span></span>

<a name="implementing"></a>   
## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="fb60f-104">Implémentation d’une propriété de dépendance de type collection</span><span class="sxs-lookup"><span data-stu-id="fb60f-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="fb60f-105">Pour une propriété de dépendance en général, le modèle d’implémentation que vous suivez est que vous définissez un [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] wrapper de propriété, où cette propriété est soutenue par un <xref:System.Windows.DependencyProperty> identificateur au lieu d’un champ ou une autre construction.</span><span class="sxs-lookup"><span data-stu-id="fb60f-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="fb60f-106">Vous devez suivre ce même modèle quand vous implémentez une propriété de type collection.</span><span class="sxs-lookup"><span data-stu-id="fb60f-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="fb60f-107">Toutefois, une propriété de type collection présente une complexité au modèle chaque fois que le type qui est inclu dans la collection est lui-même une <xref:System.Windows.DependencyObject> ou <xref:System.Windows.Freezable> classe dérivée.</span><span class="sxs-lookup"><span data-stu-id="fb60f-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="initializing-the-collection-beyond-the-default-value"></a><span data-ttu-id="fb60f-108">Initialisation de la collection au-delà de la valeur par défaut</span><span class="sxs-lookup"><span data-stu-id="fb60f-108">Initializing the Collection Beyond the Default Value</span></span>  
 <span data-ttu-id="fb60f-109">Quand vous créez une propriété de dépendance, vous ne spécifiez pas sa valeur par défaut comme étant la valeur de champ initiale.</span><span class="sxs-lookup"><span data-stu-id="fb60f-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="fb60f-110">Vous la spécifiez à l’aide des métadonnées de la propriété de dépendance.</span><span class="sxs-lookup"><span data-stu-id="fb60f-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="fb60f-111">Si votre propriété est un type référence, la valeur par défaut spécifiée dans les métadonnées de la propriété de dépendance n’est pas une valeur par défaut par instance, mais bien une valeur par défaut qui s’applique à toutes les instances du type.</span><span class="sxs-lookup"><span data-stu-id="fb60f-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; instead it is a default value that applies to all instances of the type.</span></span> <span data-ttu-id="fb60f-112">Ainsi, vous devez prendre garde de ne pas utiliser la collection statique unique définie par les métadonnées de la propriété de type collection comme valeur par défaut active pour les instances créées de votre type.</span><span class="sxs-lookup"><span data-stu-id="fb60f-112">Therefore you must be careful to not use the singular static collection defined by the collection property metadata as the working default value for newly created instances of your type.</span></span> <span data-ttu-id="fb60f-113">À la place, vous devez vous assurer que vous affectez délibérément à la valeur de collection une collection (instance) unique dans le cadre de votre logique de constructeur de classe.</span><span class="sxs-lookup"><span data-stu-id="fb60f-113">Instead, you must make sure that you deliberately set the collection value to a unique (instance) collection as part of your class constructor logic.</span></span> <span data-ttu-id="fb60f-114">Sinon, vous créerez une classe singleton involontaire.</span><span class="sxs-lookup"><span data-stu-id="fb60f-114">Otherwise you will have created an unintentional singleton class.</span></span>

## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="fb60f-129">Signalement des modifications des valeurs de liaison des propriétés de collection</span><span class="sxs-lookup"><span data-stu-id="fb60f-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="fb60f-130">Une propriété de collection qui est elle-même une propriété de dépendance ne signale pas automatiquement les modifications apportées à ses sous-propriétés.</span><span class="sxs-lookup"><span data-stu-id="fb60f-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="fb60f-131">Si vous créez des liaisons au sein d’une collection, cela peut empêcher la liaison de signaler les modifications et, de ce fait, invalider certains scénarios de liaison de données.</span><span class="sxs-lookup"><span data-stu-id="fb60f-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="fb60f-132">Toutefois, si vous utilisez le type de collection <xref:System.Windows.FreezableCollection%601> comme type de collection, puis les modifications de sous-propriétés apportées aux éléments contenus dans la collection sont correctement signalées et la liaison fonctionne comme prévu.</span><span class="sxs-lookup"><span data-stu-id="fb60f-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="fb60f-133">Pour activer la liaison de sous-propriétés dans une collection d’objets de dépendance, créez la propriété de collection en tant que type <xref:System.Windows.FreezableCollection%601>, avec une contrainte de type pour cette collection à toute <xref:System.Windows.DependencyObject> classe dérivée.</span><span class="sxs-lookup"><span data-stu-id="fb60f-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
