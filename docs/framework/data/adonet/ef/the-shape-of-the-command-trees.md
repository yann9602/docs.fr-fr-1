 <span data-ttu-id="a905f-121">Les arborescences de commandes d'entrée sont exprimées par rapport au modèle conceptuel, qui prend en charge des propriétés de navigation, des associations entre entités et l'héritage.</span><span class="sxs-lookup"><span data-stu-id="a905f-121">Input command trees are expressed against the conceptual model, which supports navigation properties, associations among entities, and inheritance.</span></span> <span data-ttu-id="a905f-122">Les arborescences de commandes de sortie sont exprimées par rapport au modèle de stockage.</span><span class="sxs-lookup"><span data-stu-id="a905f-122">Output command trees are expressed against the storage model.</span></span> <span data-ttu-id="a905f-123">Les arborescences de commandes d'entrée vous permettent de projeter des collections imbriquées, contrairement aux arborescences de commandes de sortie.</span><span class="sxs-lookup"><span data-stu-id="a905f-123">Input command trees allow you to project nested collections, but output command trees do not.</span></span>
