<span data-ttu-id="931fc-103">Le magasin d’instances de workflow SQL exécute une tâche interne qui se réveille régulièrement et détecte les instances de workflow exécutables ou activables dans la base de données de persistance.</span><span class="sxs-lookup"><span data-stu-id="931fc-103">The SQL Workflow Instance Store runs an internal task that periodically wakes up and detects runnable or activatable workflow instances in the persistence database.</span></span> <span data-ttu-id="931fc-104">En cas de détection d'une instance de workflow exécutable, il avertit l'hôte de workflow capable d'activer l'instance.</span><span class="sxs-lookup"><span data-stu-id="931fc-104">If it finds a runnable workflow instance, it notifies the workflow host that is capable of activating the instance.</span></span> <span data-ttu-id="931fc-105">En cas de détection d'une instance de workflow activable, il avertit un hôte générique qui active un hôte de workflow qui, à son tour, exécute l'instance de workflow.</span><span class="sxs-lookup"><span data-stu-id="931fc-105">If the instance store finds an activatable workflow instance, it notifies a generic host that activates a workflow host, which in turn runs the workflow instance.</span></span> <span data-ttu-id="931fc-106">Les sections suivantes de cette rubrique décrivent en détail le processus d'activation d'instance.</span><span class="sxs-lookup"><span data-stu-id="931fc-106">The following sections in this topic explain the instance activation process in detail.</span></span>  
  
##  <a name="RunnableSection"></a><span data-ttu-id="931fc-107">Détection et activation d’Instances de Workflow exécutables</span><span class="sxs-lookup"><span data-stu-id="931fc-107">Detecting and Activating Runnable Workflow Instances</span></span>  
 <span data-ttu-id="931fc-108">Le magasin d’instances de Workflow SQL considère qu’une instance de workflow *exécutable* si l’instance n’est pas dans l’état suspendu ou l’état terminé et satisfait les conditions suivantes :</span><span class="sxs-lookup"><span data-stu-id="931fc-108">The SQL Workflow Instance Store considers a workflow instance *runnable* if the instance is not in the suspended state or the completed state and satisfies the following conditions:</span></span>

-   <span data-ttu-id="931fc-110">L'instance a un verrou périmé.</span><span class="sxs-lookup"><span data-stu-id="931fc-110">The instance has an expired lock on it.</span></span>  
  
-   <span data-ttu-id="931fc-111">L’instance est déverrouillée et son état est **en cours d’exécution**.</span><span class="sxs-lookup"><span data-stu-id="931fc-111">The instance is unlocked and its status is **Executing**.</span></span>  
  
 <span data-ttu-id="931fc-112">Lorsqu'il détecte une instance exécutable, le magasin d'instances de workflow SQL déclenche l'objet <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent>.</span><span class="sxs-lookup"><span data-stu-id="931fc-112">The SQL Workflow Instance Store raises the <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> when it finds a runnable instance.</span></span> <span data-ttu-id="931fc-113">Ensuite, le SqlWorkflowInstanceStore cesse de surveiller jusqu'à ce que l'objet <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> soit appelé une fois sur le magasin.</span><span class="sxs-lookup"><span data-stu-id="931fc-113">After this, the SqlWorkflowInstanceStore stops monitoring until the <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> is called once on the store.</span></span>  
  
 <span data-ttu-id="931fc-114">Un hôte de workflow qui s'est abonné à l'objet <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> et capable de charger l'instance exécute l'objet <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> par rapport au magasin d'instances pour charger l'instance en mémoire.</span><span class="sxs-lookup"><span data-stu-id="931fc-114">A workflow host that has subscribed for the <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> and capable of loading the instance executes the <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> against the instance store to load the instance into memory.</span></span> <span data-ttu-id="931fc-115">Un hôte de workflow est considéré comme capable de charger une instance de workflow si l’ordinateur hôte et l’instance ont la propriété de métadonnées **WorkflowServiceType** la même valeur.</span><span class="sxs-lookup"><span data-stu-id="931fc-115">A workflow host is considered capable of loading a workflow instance if the host and the instance have metadata property **WorkflowServiceType** set to the same value.</span></span>
