## <a name="delegates-and-their-role"></a><span data-ttu-id="9ec3d-114">Rôle des délégués</span><span class="sxs-lookup"><span data-stu-id="9ec3d-114">Delegates and Their Role</span></span>  
 <span data-ttu-id="9ec3d-115">Les délégués sont des classes qui sont communément utilisées dans [!INCLUDE[dnprdnshort](../../../includes/dnprdnshort-md.md)] pour générer des mécanismes de gestion des événements.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-115">Delegates are classes commonly used within the [!INCLUDE[dnprdnshort](../../../includes/dnprdnshort-md.md)] to build event-handling mechanisms.</span></span> <span data-ttu-id="9ec3d-116">Les délégués sont à peu près équivalents aux pointeurs de fonction communément utilisés dans [!INCLUDE[vcprvc](../../../includes/vcprvc-md.md)] et d'autres langages orientés objet.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-116">Delegates roughly equate to function pointers, commonly used in [!INCLUDE[vcprvc](../../../includes/vcprvc-md.md)] and other object-oriented languages.</span></span> <span data-ttu-id="9ec3d-117">Toutefois à la différence des pointeurs de fonction, les délégués sont orientés objet, de type sécurisé et sûrs.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-117">Unlike function pointers however, delegates are object-oriented, type-safe, and secure.</span></span> <span data-ttu-id="9ec3d-118">De plus, alors qu'un pointeur de fonction ne contient qu'une référence à une fonction spécifique, un délégué consiste en une référence à un objet et des références à une ou plusieurs méthodes de cet objet.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-118">In addition, where a function pointer contains only a reference to a particular function, a delegate consists of a reference to an object, and references to one or more methods within the object.</span></span>  
  
 <span data-ttu-id="9ec3d-119">Ce modèle d’événement utilise *délégués* pour lier les événements aux méthodes servant à les gérer.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-119">This event model uses *delegates* to bind events to the methods that are used to handle them.</span></span> <span data-ttu-id="9ec3d-120">Le délégué permet aux autres classes de s'inscrire pour la notification d'événement en spécifiant une méthode de gestionnaire.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-120">The delegate enables other classes to register for event notification by specifying a handler method.</span></span> <span data-ttu-id="9ec3d-121">Quand l'événement se produit, le délégué appelle la méthode liée.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-121">When the event occurs, the delegate calls the bound method.</span></span> <span data-ttu-id="9ec3d-122">Pour plus d’informations sur la façon de définir les délégués, consultez [événements](../../../docs/standard/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="9ec3d-122">For more information about how to define delegates, see [Events](../../../docs/standard/events/index.md).</span></span>  
  
 <span data-ttu-id="9ec3d-123">Les délégués peuvent être liés à une ou plusieurs méthodes (ou multidiffusion).</span><span class="sxs-lookup"><span data-stu-id="9ec3d-123">Delegates can be bound to a single method or to multiple methods, referred to as multicasting.</span></span> <span data-ttu-id="9ec3d-124">Quand vous créez un délégué pour un événement, vous (ou le Concepteur Windows Forms) créez généralement un événement multidiffusion.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-124">When creating a delegate for an event, you (or the Windows Forms Designer) typically create a multicast event.</span></span> <span data-ttu-id="9ec3d-125">Il existe cependant une exception à cette règle : quand un événement déclenche une procédure particulière (par exemple, l'affichage d'une boîte de dialogue) qui logiquement ne se répéterait pas plusieurs fois par événement.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-125">A rare exception might be an event that results in a specific procedure (such as displaying a dialog box) that would not logically repeat multiple times per event.</span></span> <span data-ttu-id="9ec3d-126">Pour plus d’informations sur la création d’un délégué multicast, consultez [Comment : combiner des délégués (délégués Multicast)](~/docs/csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md).</span><span class="sxs-lookup"><span data-stu-id="9ec3d-126">For information about how to create a multicast delegate, see [How to: Combine Delegates (Multicast Delegates)](~/docs/csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md).</span></span>  
  
 <span data-ttu-id="9ec3d-127">Un délégué multidiffusion tient à jour une liste d'appel des méthodes auxquelles il est lié.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-127">A multicast delegate maintains an invocation list of the methods it is bound to.</span></span> <span data-ttu-id="9ec3d-128">Celui-ci prend en charge une méthode <xref:System.Delegate.Combine%2A> pour ajouter une méthode à la liste d'appel et une méthode <xref:System.Delegate.Remove%2A> pour la supprimer.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-128">The multicast delegate supports a <xref:System.Delegate.Combine%2A> method to add a method to the invocation list and a <xref:System.Delegate.Remove%2A> method to remove it.</span></span>  
  
 <span data-ttu-id="9ec3d-129">Quand un événement est enregistré par l'application, le contrôle le déclenche en appelant son délégué.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-129">When an event is recorded by the application, the control raises the event by invoking the delegate for that event.</span></span> <span data-ttu-id="9ec3d-130">Le délégué appelle à son tour la méthode liée.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-130">The delegate in turn calls the bound method.</span></span> <span data-ttu-id="9ec3d-131">Dans le cas le plus courant (délégué multidiffusion), le délégué appelle à tour de rôle chaque méthode liée de la liste d'appel, ce qui produit une notification un à plusieurs.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-131">In the most common case (a multicast delegate) the delegate calls each bound method in the invocation list in turn, which provides a one-to-many notification.</span></span> <span data-ttu-id="9ec3d-132">Cette stratégie signifie que le contrôle n'a pas besoin de tenir à jour une liste des objets cible pour la notification d'événement car le délégué gère totalement l'inscription et la notification.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-132">This strategy means that the control does not need to maintain a list of target objects for event notification—the delegate handles all registration and notification.</span></span>  
  
 <span data-ttu-id="9ec3d-133">Les délégués permettent également de lier plusieurs événements à la même méthode et par conséquent de produire une notification plusieurs-à-un.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-133">Delegates also enable multiple events to be bound to the same method, allowing a many-to-one notification.</span></span> <span data-ttu-id="9ec3d-134">Par exemple, un événement button-click et un événement menu-command–click peuvent tous deux appeler le même délégué, lequel appelle à son tour une méthode unique pour qu'elle prenne en charge ces deux événements distincts de la même façon.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-134">For example, a button-click event and a menu-command–click event can both invoke the same delegate, which then calls a single method to handle these separate events the same way.</span></span>  
  
 <span data-ttu-id="9ec3d-135">Le mécanisme de liaison utilisé avec les délégués est dynamique : un délégué peut être lié au moment de l'exécution à n'importe quelle méthode dont la signature correspond à celle du gestionnaire d'événements.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-135">The binding mechanism used with delegates is dynamic: a delegate can be bound at run time to any method whose signature matches that of the event handler.</span></span> <span data-ttu-id="9ec3d-136">Cette fonctionnalité vous permet de configurer ou de modifier la méthode de liaison en fonction d’une condition et d’attacher dynamiquement un gestionnaire d’événements à un contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ec3d-136">With this feature, you can set up or change the bound method depending on a condition and to dynamically attach an event handler to a control.</span></span>
