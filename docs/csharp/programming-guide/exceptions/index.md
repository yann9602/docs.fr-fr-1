<span data-ttu-id="5823a-103">Les fonctionnalités de gestion des exceptions du langage C# vous aident à gérer les situations inattendues ou exceptionnelles qui se produisent lorsqu’un programme est en cours d’exécution.</span><span class="sxs-lookup"><span data-stu-id="5823a-103">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="5823a-104">La gestion des exceptions utilise les mots clés `try`, `catch` et `finally` pour tenter des actions susceptibles de ne pas réussir, pour gérer les défaillances lorsque vous pensez que c’est justifié et pour nettoyer ensuite les ressources.</span><span class="sxs-lookup"><span data-stu-id="5823a-104">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it is reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="5823a-105">Les exceptions peuvent être générées par le common language runtime (CLR), par .NET Framework ou des bibliothèques tierces, ou par le code de l’application.</span><span class="sxs-lookup"><span data-stu-id="5823a-105">Exceptions can be generated by the common language runtime (CLR), by the .NET Framework or any third-party libraries, or by application code.</span></span> <span data-ttu-id="5823a-106">Les exceptions sont créées avec le mot clé `throw`.</span><span class="sxs-lookup"><span data-stu-id="5823a-106">Exceptions are created by using the `throw` keyword.</span></span>  
  
 <span data-ttu-id="5823a-107">Dans de nombreux cas, une exception peut être levée non pas par une méthode appelée directement par votre code, mais par une autre méthode plus loin dans la pile des appels.</span><span class="sxs-lookup"><span data-stu-id="5823a-107">In many cases, an exception may be thrown not by a method that your code has called directly, but by another method further down in the call stack.</span></span> <span data-ttu-id="5823a-108">Dans ce cas, le CLR déroule la pile à la recherche d’une méthode avec un bloc `catch` pour le type d’exception concerné et exécute le premier bloc `catch` de ce type qu’il trouve.</span><span class="sxs-lookup"><span data-stu-id="5823a-108">When this happens, the CLR will unwind the stack, looking for a method with a `catch` block for the specific exception type, and it will execute the first such `catch` block that if finds.</span></span> <span data-ttu-id="5823a-109">S’il ne trouve pas de bloc `catch` dans la pile des appels, il termine le processus et affiche un message à l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="5823a-109">If it finds no appropriate `catch` block anywhere in the call stack, it will terminate the process and display a message to the user.</span></span>  
  
 <span data-ttu-id="5823a-110">Dans cet exemple, une méthode teste la division par zéro et intercepte l’erreur.</span><span class="sxs-lookup"><span data-stu-id="5823a-110">In this example, a method tests for division by zero and catches the error.</span></span> <span data-ttu-id="5823a-111">Sans la gestion des exceptions, ce programme se terminerait avec une erreur **DivideByZeroException non gérée**.</span><span class="sxs-lookup"><span data-stu-id="5823a-111">Without the exception handling, this program would terminate with a **DivideByZeroException was unhandled** error.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="5823a-113">Vue d’ensemble des exceptions</span><span class="sxs-lookup"><span data-stu-id="5823a-113">Exceptions Overview</span></span>  
 <span data-ttu-id="5823a-114">Les exceptions ont les propriétés suivantes :</span><span class="sxs-lookup"><span data-stu-id="5823a-114">Exceptions have the following properties:</span></span>  
  
-   <span data-ttu-id="5823a-115">Les exceptions sont des types qui dérivent tous en définitive de `System.Exception`.</span><span class="sxs-lookup"><span data-stu-id="5823a-115">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>  
  
-   <span data-ttu-id="5823a-116">Utilisez un bloc `try` autour des instructions qui peuvent lever des exceptions.</span><span class="sxs-lookup"><span data-stu-id="5823a-116">Use a `try` block around the statements that might throw exceptions.</span></span>  
  
-   <span data-ttu-id="5823a-117">Dès qu’une exception se produit dans le bloc `try`, le flux de contrôle passe immédiatement au premier gestionnaire d’exceptions associé présent dans la pile des appels.</span><span class="sxs-lookup"><span data-stu-id="5823a-117">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="5823a-118">En C#, le mot clé `catch` est utilisé pour définir un gestionnaire d’exceptions.</span><span class="sxs-lookup"><span data-stu-id="5823a-118">In C#, the `catch` keyword is used to define an exception handler.</span></span>  
  
-   <span data-ttu-id="5823a-119">Si aucun gestionnaire d’exceptions n’est présent pour une exception donnée, le programme s’arrête avec un message d’erreur.</span><span class="sxs-lookup"><span data-stu-id="5823a-119">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>  
  
-   <span data-ttu-id="5823a-120">N’interceptez pas d’exception si vous ne pouvez pas la gérer tout en laissant l’application dans un état connu.</span><span class="sxs-lookup"><span data-stu-id="5823a-120">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="5823a-121">Si vous interceptez `System.Exception`, levez-la de nouveau avec le mot clé `throw` à la fin du bloc `catch`.</span><span class="sxs-lookup"><span data-stu-id="5823a-121">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>  
  
-   <span data-ttu-id="5823a-122">Si un bloc `catch` définit une variable d’exception, vous pouvez l’utiliser pour obtenir plus d’informations sur le type d’exception qui s’est produit.</span><span class="sxs-lookup"><span data-stu-id="5823a-122">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>  
  
-   <span data-ttu-id="5823a-123">Les exceptions peuvent être générées explicitement par un programme avec le mot clé `throw`.</span><span class="sxs-lookup"><span data-stu-id="5823a-123">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>  
  
-   <span data-ttu-id="5823a-124">Les objets Exception contiennent des informations détaillées sur l'erreur, telles que l'état de la pile des appels et une description du texte de l'erreur.</span><span class="sxs-lookup"><span data-stu-id="5823a-124">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>  
  
-   <span data-ttu-id="5823a-125">Le code qui se trouve dans un bloc `finally` est exécuté même si une exception est levée.</span><span class="sxs-lookup"><span data-stu-id="5823a-125">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="5823a-126">Utilisez un bloc `finally` pour libérer des ressources, par exemple pour fermer tous les flux ou fichiers qui ont été ouverts dans le bloc `try`.</span><span class="sxs-lookup"><span data-stu-id="5823a-126">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>  
  
-   <span data-ttu-id="5823a-127">Les exceptions gérées dans .NET Framework sont implémentées au-dessus du mécanisme de gestion structurée des exceptions de Win32.</span><span class="sxs-lookup"><span data-stu-id="5823a-127">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="5823a-128">Pour plus d’informations, consultez les pages [Gestion structurée des exceptions (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) et [Cours intensif sur la gestion structurée des exceptions de Win32](http://go.microsoft.com/fwlink/?LinkId=119654).</span><span class="sxs-lookup"><span data-stu-id="5823a-128">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://go.microsoft.com/fwlink/?LinkId=119654).</span></span>
