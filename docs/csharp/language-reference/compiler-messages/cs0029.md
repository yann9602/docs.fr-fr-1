# <a name="compiler-error-cs0029"></a><span data-ttu-id="27142-102">Erreur du compilateur CS0029</span><span class="sxs-lookup"><span data-stu-id="27142-102">Compiler Error CS0029</span></span>
<span data-ttu-id="27142-103">Impossible de convertir implicitement le type 'type' en 'type'</span><span class="sxs-lookup"><span data-stu-id="27142-103">Cannot implicitly convert type 'type' to 'type'</span></span>  
  
 <span data-ttu-id="27142-104">Le compilateur requiert une conversion explicite.</span><span class="sxs-lookup"><span data-stu-id="27142-104">The compiler requires an explicit conversion.</span></span> <span data-ttu-id="27142-105">Par exemple, il se peut que vous deviez effectuer un cast sur une valeur r en type identique à celui d’une valeur l.</span><span class="sxs-lookup"><span data-stu-id="27142-105">For example, you may need to cast an r-value to be the same type as an l-value.</span></span> <span data-ttu-id="27142-106">Vous pouvez aussi avoir à fournir des routines de conversion pour prendre en charge certaines surcharges d’opérateur.</span><span class="sxs-lookup"><span data-stu-id="27142-106">Or, you must provide conversion routines to support certain operator overloads.</span></span>  
  
 <span data-ttu-id="27142-107">Les conversions doivent se produire lors de l’assignation d’une variable d’un type à une variable d’un autre type.</span><span class="sxs-lookup"><span data-stu-id="27142-107">Conversions must occur when assigning a variable of one type to a variable of a different type.</span></span> <span data-ttu-id="27142-108">Lors de l’assignation entre des variables de types différents, le compilateur doit convertir le type situé à droite de l’opérateur d’assignation dans le type situé à gauche de ce même opérateur.</span><span class="sxs-lookup"><span data-stu-id="27142-108">When making an assignment between variables of different types, the compiler must convert the type on the right-hand side of the assignment operator to the type on the left-hand side of the assignment operator.</span></span> <span data-ttu-id="27142-109">Prenez le code ci-dessous :</span><span class="sxs-lookup"><span data-stu-id="27142-109">Take the following the code:</span></span>

```csharp  
int i = 50;  
long lng = 100;  
i = lng;  
```  
  
 <span data-ttu-id="27142-110">`i = lng;` effectue une assignation, mais les types de données des variables situées de part et d’autre de l’opérateur d’assignation ne correspondent pas.</span><span class="sxs-lookup"><span data-stu-id="27142-110">`i = lng;` makes an assignment, but the data types of the variables on the left and right-hand side of the assignment operator don't match.</span></span> <span data-ttu-id="27142-111">Avant de procéder à l’assignation, le compilateur convertit implicitement la variable `lng`, qui est de type long, en type int.</span><span class="sxs-lookup"><span data-stu-id="27142-111">Before making the assignment the compiler is implicitly converting the variable `lng`, which is of type long, to an int.</span></span> <span data-ttu-id="27142-112">Cela est implicite, car aucun code n’a indiqué explicitement au compilateur d’effectuer cette conversion.</span><span class="sxs-lookup"><span data-stu-id="27142-112">This is implicit because no code explicitly instructed the compiler to perform this conversion.</span></span> <span data-ttu-id="27142-113">Il s’agit néanmoins d’une conversion restrictive et c’est là où se pose un problème ; en effet, le compilateur n’autorise pas les conversions restrictives implicites, car il existe un risque potentiel de perte de données.</span><span class="sxs-lookup"><span data-stu-id="27142-113">The problem with this code is that this is considered a narrowing conversion, and the compiler does not allow implicit narrowing conversions because there could be a potential loss of data.</span></span>  
  
 <span data-ttu-id="27142-114">Une conversion restrictive existe lors de la conversion en un type de données qui occupe moins d’espace de stockage en mémoire que le type de données à partir duquel vous effectuez la conversion.</span><span class="sxs-lookup"><span data-stu-id="27142-114">A narrowing conversion exists when converting to a data type that occupies less storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="27142-115">Par exemple, la conversion d’un type long en type int est considérée comme une conversion restrictive.</span><span class="sxs-lookup"><span data-stu-id="27142-115">For example, converting a long to an int would be considered a narrowing conversion.</span></span> <span data-ttu-id="27142-116">Le type long occupe 8 octets de mémoire tandis que le type int en occupe 4.</span><span class="sxs-lookup"><span data-stu-id="27142-116">A long occupies 8 bytes of memory while an int occupies 4 bytes.</span></span> <span data-ttu-id="27142-117">Pour voir dans quelles conditions survient la perte de données, prenez l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="27142-117">To see how data loss can occur, consider the following sample:</span></span>  
  
```csharp  
int i = 50;  
long lng = 3147483647;  
i = lng;  
```  
  
 <span data-ttu-id="27142-118">La variable `lng` contient maintenant une valeur qui ne peut être stockée dans la variable `i` du fait de sa trop grande taille.</span><span class="sxs-lookup"><span data-stu-id="27142-118">The variable `lng` now contains a value that cannot be stored in the variable `i` because it is too large.</span></span> <span data-ttu-id="27142-119">Si vous deviez convertir cette valeur en type int, vous perdriez certaines données et la valeur convertie ne serait pas la même que la valeur avant la conversion.</span><span class="sxs-lookup"><span data-stu-id="27142-119">If we were to convert this value to an int type we would be losing some of our data and the converted value would not be the same as the value before the conversion.</span></span>  
  
 <span data-ttu-id="27142-120">Une conversion étendue est le contraire d’une conversion restrictive.</span><span class="sxs-lookup"><span data-stu-id="27142-120">A widening conversion would be the opposite of a narrowing conversion.</span></span> <span data-ttu-id="27142-121">Dans ce cas, vous effectuez une conversion en un type de données qui occupe plus d’espace de stockage en mémoire que le type de données à partir duquel vous effectuez la conversion.</span><span class="sxs-lookup"><span data-stu-id="27142-121">With widening conversions, we are converting to a data type that occupies more storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="27142-122">En voici un exemple :</span><span class="sxs-lookup"><span data-stu-id="27142-122">Here is an example of a widening conversion:</span></span>

```csharp  
lng = i;  
```  
  
 <span data-ttu-id="27142-123">Notez la différence entre cet exemple de code et le premier.</span><span class="sxs-lookup"><span data-stu-id="27142-123">Notice the difference between this code sample and the first.</span></span> <span data-ttu-id="27142-124">Cette fois, la variable `lng` se trouve à gauche de l’opérateur d’assignation, de sorte qu’elle est la cible de votre assignation.</span><span class="sxs-lookup"><span data-stu-id="27142-124">This time the variable `lng` is on the left-hand side of the assignment operator, so it is the target of our assignment.</span></span> <span data-ttu-id="27142-125">Pour que l’assignation puisse être effectuée, le compilateur doit implicitement convertir la variable `i`, qui est de type int, en type long.</span><span class="sxs-lookup"><span data-stu-id="27142-125">Before the assignment can be made, the compiler must implicitly convert the variable `i`, which is of type int, to type long.</span></span> <span data-ttu-id="27142-126">Il s’agit d’une conversion étendue puisqu’elle s’effectue à partir d’un type qui occupe 4 octets de mémoire (int) vers un type qui en occupe 8 (long).</span><span class="sxs-lookup"><span data-stu-id="27142-126">This is a widening conversion since we are converting from a type that occupies 4 bytes of memory (an int) to a type that occupies 8 bytes of memory (a long).</span></span> <span data-ttu-id="27142-127">Les conversions étendues implicites sont autorisées, car il n’existe aucun risque potentiel de perte de données.</span><span class="sxs-lookup"><span data-stu-id="27142-127">Implicit widening conversions are allowed because there is no potential loss of data.</span></span> <span data-ttu-id="27142-128">Toute valeur pouvant être stockée en type int peut l’être aussi en long.</span><span class="sxs-lookup"><span data-stu-id="27142-128">Any value that can be stored in an int can also be stored in a long.</span></span>  
  
 <span data-ttu-id="27142-129">Les conversions restrictives implicites n’étant pas autorisées, il faut donc, pour compiler ce code, convertir explicitement le type de données.</span><span class="sxs-lookup"><span data-stu-id="27142-129">We know that implicit narrowing conversions are not allowed, so to be able to compile this code we need to explicitly convert the data type.</span></span> <span data-ttu-id="27142-130">Les conversions explicites sont effectuées à l’aide du cast.</span><span class="sxs-lookup"><span data-stu-id="27142-130">Explicit conversions are done using casting.</span></span> <span data-ttu-id="27142-131">Le cast est le terme utilisé en C# pour décrire la conversion d’un type de données en un autre.</span><span class="sxs-lookup"><span data-stu-id="27142-131">Casting is the term used in C# to describe converting one data type to another.</span></span> <span data-ttu-id="27142-132">Pour obtenir le code à compiler, vous devez utiliser la syntaxe suivante :</span><span class="sxs-lookup"><span data-stu-id="27142-132">To get the code to compile we would need to use the following syntax:</span></span>

  
 <span data-ttu-id="27142-133">La troisième ligne de code indique au compilateur de convertir explicitement la variable `lng`, qui est de type long, en type int avant de procéder à l’assignation.</span><span class="sxs-lookup"><span data-stu-id="27142-133">The third line of code tells the compiler to explicitly convert the variable `lng`, which is of type long, to an int before making the assignment.</span></span> <span data-ttu-id="27142-134">N’oubliez pas qu’avec une conversion restrictive, il existe un risque de perte de données.</span><span class="sxs-lookup"><span data-stu-id="27142-134">Remember that with a narrowing conversion, there is a potential loss of data.</span></span> <span data-ttu-id="27142-135">Les conversions restrictives doivent être utilisées avec précaution car, même si la compilation est effectuée, des résultats inattendus peuvent se produire au moment de l’exécution.</span><span class="sxs-lookup"><span data-stu-id="27142-135">Narrowing conversions should be used with caution and even though the code will compile you may get unexpected results at run-time.</span></span>
