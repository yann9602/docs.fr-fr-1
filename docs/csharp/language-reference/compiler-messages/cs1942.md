# <a name="compiler-error-cs1942"></a><span data-ttu-id="ca9bc-102">Erreur du compilateur CS1942</span><span class="sxs-lookup"><span data-stu-id="ca9bc-102">Compiler Error CS1942</span></span>
<span data-ttu-id="ca9bc-103">Le type de l’expression dans la clause 'clause' est incorrect.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-103">The type of the expression in the 'clause' clause is incorrect.</span></span> <span data-ttu-id="ca9bc-104">L’inférence de type a échoué lors de l’appel à la méthode 'method'.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-104">Type inference failed in the call to 'method'.</span></span>  
  
 <span data-ttu-id="ca9bc-105">Cette erreur est habituellement générée quand un type explicite incorrect a été attribué à la variable de portée.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-105">This error is typically generated when the range variable has been given an incorrect explicit type.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="ca9bc-106">Pour corriger cette erreur</span><span class="sxs-lookup"><span data-stu-id="ca9bc-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="ca9bc-107">Si la variable de portée est explicitement typée, vérifiez que le type est identique à celui des éléments de la collection qu’il itère ou que le type est implicitement convertible à partir du type de ces éléments.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-107">If the range variable is explicitly typed, make sure that the type is either the same as, or implicitly convertible from, the type of the elements in the collection it iterates.</span></span> <span data-ttu-id="ca9bc-108">Si la variable de portée est précédée du mot clé `var`, supprimez `var`.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-108">If the range variable is preceded with the `var` keyword, remove `var`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="ca9bc-109">Exemple</span><span class="sxs-lookup"><span data-stu-id="ca9bc-109">Example</span></span>  
 <span data-ttu-id="ca9bc-110">Le code suivant génère l’erreur CS1942 :</span><span class="sxs-lookup"><span data-stu-id="ca9bc-110">The following code generates CS1942:</span></span>  
  
```csharp  
// cs1942.cs  
class Program  
    {  
        static void Main(string[] args)  
        {  
            var x = from var i in Enumerable.Range(1, 100) // CS1949  
                    select i; //CS1942  
        }  
    }  
```  
  
 <span data-ttu-id="ca9bc-111">CS1942 est lié à CS1949, car l’utilisation de `var` avec une variable de portée entraîne l’échec de l’opération `Cast<T>` sous-jacente, puisque `var` n’est pas un type.</span><span class="sxs-lookup"><span data-stu-id="ca9bc-111">CS1942 is related to CS1949 because the use of `var` with a range variable causes the underlying `Cast<T>` operation to fail because `var` is not a type.</span></span>
