# <a name="compiler-error-cs0686"></a><span data-ttu-id="ef7ba-102">Erreur du compilateur CS0686</span><span class="sxs-lookup"><span data-stu-id="ef7ba-102">Compiler Error CS0686</span></span>
<span data-ttu-id="ef7ba-103">L’accesseur 'accessor' ne peut pas implémenter le membre d’interface 'member' pour le type 'type'.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-103">Accessor 'accessor' cannot implement interface member 'member' for type 'type'.</span></span> <span data-ttu-id="ef7ba-104">Utilisez une implémentation d’interface explicite.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-104">Use an explicit interface implementation.</span></span>  
  
 <span data-ttu-id="ef7ba-105">Suggestion : cette erreur peut se produire lors de l’implémentation d’une interface qui contient des noms de méthodes en conflit avec les méthodes générées automatiquement et associées à une propriété ou à un événement.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-105">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="ef7ba-106">Les méthodes get/set des propriétés sont générées sous la forme get_property et set_property, et les méthodes add/remove des événements sont générées sous la forme add_event et remove_event.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-106">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="ef7ba-107">Si une interface contient l’une ou l’autre de ces méthodes, un conflit se produit.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-107">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="ef7ba-108">Pour éviter cette erreur, implémentez les méthodes à l’aide d’une implémentation d’interface explicite.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-108">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="ef7ba-109">Pour ce faire, spécifiez la fonction comme suit :</span><span class="sxs-lookup"><span data-stu-id="ef7ba-109">To do this, specify the function as:</span></span>  
  
```csharp  
Interface.get_property() { /* */ }

```  
  
## <a name="example"></a><span data-ttu-id="ef7ba-110">Exemple</span><span class="sxs-lookup"><span data-stu-id="ef7ba-110">Example</span></span>  
 <span data-ttu-id="ef7ba-111">L’exemple suivant génère l’erreur CS0686 :</span><span class="sxs-lookup"><span data-stu-id="ef7ba-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="ef7ba-112">Exemple</span><span class="sxs-lookup"><span data-stu-id="ef7ba-112">Example</span></span>  
 <span data-ttu-id="ef7ba-113">Cette erreur peut également se produire quand vous déclarez des événements.</span><span class="sxs-lookup"><span data-stu-id="ef7ba-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="ef7ba-114">La construction d’événements génère automatiquement les méthodes `add_``event` et `remove_``event`, qui sont susceptibles d’être en conflit avec les méthodes portant le même nom dans une interface, comme dans l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="ef7ba-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
